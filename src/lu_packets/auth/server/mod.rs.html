28/**
29    Provides username and password to authenticate the client.
30
31    Also provides system stats for analytics.
32
33    ### Trigger
34    Receipt of [`Handshake`](crate::general::client::Handshake) with a [`service_id`](crate::general::client::Handshake`::service_id) of [`ServiceId::Auth`].
35
36    ### Handling
37    Look up the username and a hashed form of the password in the database. If there is a match, generate a session key to be used as an auth token when the client connects to world servers, and save it to DB. Generate session keys in a way that makes it impossible for an attacker to guess a session key with non-neglible probability. Specifically, this means that using an incrementing number or a non-cryptographically-secure pseudo-random number generator for session keys is ***not*** secure.
38
39    In addition, determine the address of a char server to redirect the client to.
40
41    ### Response
42    Respond with [`LoginResponse`](super::client::LoginResponse), using an appropriate variant to indicate the lookup status, passing the session key and redirect address if successful.
43
44    ### Notes
45    The password is provided in plain text. **Don't** save this password to the database unprocessed, as this constitutes a **security hazard**. Hash and salt it using a strong cryptographic hash function before saving it.
46*/
47#[derive(Debug, Deserialize, PartialEq, Serialize)]
48pub struct LoginRequest {
49    /// The client's user name.
50    pub username: LuWString33,
51    /// The client's password.
52    pub password: LuWString41,
53    /// The client's locale.
54    pub locale_id: u16,
55    /// The client's operating system.
56    pub client_os: ClientOs,
57    /// Stats about the computer the client is running on.
58    pub computer_stats: ComputerStats,
59}
